# PLAYBOOK · 做网站流程

## Step 1: 聊（1小时）

视频通话。了解三件事：
- 你是谁、你做什么
- 你现在的网上存在是什么样的
- 你想要一个什么样的地方

### Key Questions
1. 你现在在网上有哪些地方？（B站、微博、Instagram...）
2. 你觉得缺什么？
3. 如果有一个人第一次想了解你——你希望他看到什么？
4. 你最喜欢什么颜色/氛围？（发几个参考网站或图片）
5. 你的内容是什么样的？（发链接）

## Step 2: 设计（3天）

从对话里提取方向——不是从模板库里选。
- 颜色：他说了什么颜色？他的内容是什么氛围？
- 排版：干净的无衬线（现代、直接）vs 有温度的衬线（安静、叙事）
- 结构：单页滚动 vs 多页导航——取决于内容量
- 动效：有（Framer Motion）但不多——呼吸感，不炫技

### 设计原则
- 围绕他——不是围绕"好看"
- 快——不堆功能
- 干净——每个元素有存在的理由
- 手机也要好——一半以上的访客在手机上

## Step 3: 开发（2天）

Next.js + Tailwind + Framer Motion + Vercel。

- 响应式（手机/平板/桌面）
- SEO（title/description/OG）
- 速度（图片优化、懒加载）
- 域名（客户自己买，我们帮配置）

## Step 4: 上线

代码推到 GitHub → Vercel 自动部署。客户拿到一个 URL。他的网上空间上线了。
