# 032 — Templates

Reusable documents for every project. Copy. Fill in. Send.

## Template Library

| Template | Usage |
|----------|-------|
| `Proposal.md` | Client proposal with tier, timeline, and price |
| `Invoice.md` | Payment request |
| `Welcome_Pack.md` | Post-deposit onboarding guide |
| `Interview_Notes.md` | Structured note-taking during discovery |
| `Design_Direction.md` | One-page design direction document |
| `Client_Handoff.md` | Launch day message template |
| `Check_In_1_Week.md` | Post-launch check-in |
| `Check_In_1_Month.md` | Long-term follow-up |
| `Case_Study_Request.md` | Asking permission to publish |
| `Referral_Ask.md` | Light-touch referral request |

## Usage

1. Open the template
2. Copy the content
3. Replace [bracketed] fields with client-specific info
4. Personalize — never send a template without adapting it
5. Send

## Rule

**Templates are starting points, not final drafts.** Every template should be adapted to the specific client. They save time on structure — not on humanity.
