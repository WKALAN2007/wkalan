# 017 — Project Workflow

## From "Hello" to "Holy Shit This Is Me"

Every WKALAN project follows the same 10 stages. Nothing is skipped. Nothing is rushed.

---

## The 10 Stages

```
01. INQUIRE     ← First contact. Qualify. Book discovery call.
02. DISCOVER    ← Deep interview(s). Record. Transcribe. Extract.
03. DIRECT      ← Find the 3 words. Create design direction.
04. STRUCTURE   ← Information architecture. Section order. Narrative arc.
05. DESIGN      ← Visual design. Typography. Color. Motion language.
06. BUILD       ← Code. Next.js. framer-motion. Responsive.
07. REVIEW      ← Client feedback. Revisions. Polish.
08. LAUNCH      ← Deploy. Announce. Celebrate.
09. SUPPORT     ← Post-launch care. Fixes. Updates.
10. REFLECT     ← Case study. Document learnings. Improve the system.
```

---

## Stage Details

### 01. INQUIRE
**Duration:** < 24 hours to respond
**Goal:** Qualify the client. Book the call.
**Output:** Calendly booking confirmed.
**Template:** `032_Templates/01_First_Contact_Response.md`

### 02. DISCOVER
**Duration:** 1–3 sessions (depending on tier)
**Goal:** Understand this person more deeply than anyone online.
**Output:** Interview recording + transcript + keyword extraction.
**Framework:** `007 Creator Interview Framework.md`

### 03. DIRECT
**Duration:** 1–2 days
**Goal:** Translate understanding into visual direction.
**Output:** One-page design direction document (3 keywords → Mood → Color → Type → Layout).
**Template:** `032_Templates/03_Design_Direction.md`

### 04. STRUCTURE
**Duration:** 0.5–1 day
**Goal:** Define the narrative structure.
**Output:** Section list with content outline for each section.
**Patterns:** `02_Design_System/023 Patterns/`

### 05. DESIGN
**Duration:** 2–3 days
**Goal:** Visual design of key screens.
**Output:** Homepage design + key section designs. Not every page — enough to build from.
**Tools:** Figma (or go straight to code for simpler projects).

### 06. BUILD
**Duration:** 3–6 days (depending on tier)
**Goal:** Production-ready digital identity.
**Output:** Deployed to staging. Responsive. Animated. Accessible.
**Standards:** `03_Engineering_System/`

### 07. REVIEW
**Duration:** 1–3 days
**Goal:** Client feedback. Revisions. Polish.
**Output:** Approved build.
**Rule:** Feedback is collected in one batch — not drip-fed over weeks.

### 08. LAUNCH
**Duration:** < 1 day
**Goal:** Go live.
**Output:** Production deployment. Custom domain (if applicable). Client announcement support.
**Checklist:** `02_Design_System/024 Quality Checklist.md` + `03_Engineering_System/029 Engineering Checklist.md`

### 09. SUPPORT
**Duration:** 1–4 weeks (depending on tier)
**Goal:** Post-launch stability.
**Output:** Bug fixes. Minor updates. Client confidence.

### 10. REFLECT
**Duration:** 1–2 days
**Goal:** Turn the project into proof.
**Output:** Case study (for Pro tier). Process improvements. System updates.
**Framework:** `021 Case Study Framework.md`

---

## Workflow Rules

### 1. Stages are sequential
Don't jump ahead. Don't skip. Each stage depends on the previous one.

### 2. Client sees progress daily
A screenshot. A preview link. A quick message. Never go silent.

### 3. Feedback is batched
One round of feedback per review stage. Not an endless stream of Slack messages.

### 4. The process is the product
The client's experience of being understood matters as much as the final website. Every stage communicates: *"You are in good hands."*

*Workflow is not bureaucracy. It's the structure that makes creativity reliable.*
