# 07_Playbooks

## 不是文件。是 SOP。

每一份 Playbook 回答一個問題：**「下一步我該怎麼做？」**

不用思考流程。照著做。把心力留給客戶。

---

## 結構

```
07_Playbooks/
├── 01_Discovery.md         ← 第一次接觸 → 了解客戶
├── 02_Client_Interview.md  ← 深度訪談（使用 Interview Framework）
├── 03_Research.md          ← 關鍵詞提取 → 設計研究
├── 04_Brand_Strategy.md    ← 品牌方向文件
├── 05_Sitemap.md           ← 資訊架構
├── 06_Wireframe.md         ← 線框（可跳過，直接設計）
├── 07_UI_Design.md         ← 視覺設計
├── 08_Prototype.md         ← 可互動原型
├── 09_Development.md       ← 開發（Next.js + WDS）
├── 10_QA.md                ← 品質檢查
├── 11_Launch.md            ← 上線
├── 12_Maintenance.md       ← 維護
├── 13_Case_Study.md        ← 案例撰寫
└── 14_Portfolio_Update.md  ← 更新作品集
```

---

## 每一份 Playbook 的結構

1. **Goal** — 這個階段要產出什麼
2. **Input** — 開始前需要什麼
3. **Process** — 步驟（照著做）
4. **Output** — 結束時交出什麼
5. **Quality Gate** — 怎麼知道做對了
6. **Time** — 大概要多久

---

## 使用方式

新客戶來 → 打開 `01_Discovery` → 照著做。
完成後 → 打開 `02_Client_Interview` → 照著做。
完成後 → 打開 `03_Research` → 照著做。

**不用想流程。專心想客戶。**

---

## 原則

**Playbook 是活的。** 每次做完一個專案，回來更新 Playbook。流程變順了→更新。發現更好的做法→更新。犯了一個錯→更新。

---

*Playbook 讓 WKALAN 可以複製。不是複製設計。是複製品質。*
