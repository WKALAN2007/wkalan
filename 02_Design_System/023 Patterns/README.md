# 023 — Patterns

## Philosophy

**Patterns are the grammar of layout.**

A pattern is a recurring solution to a recurring problem. Apple uses patterns. Linear uses patterns. Every great design system is built on patterns — not because they lack creativity, but because they've identified what works and committed to it.

---

## Pattern Library

### Hero Pattern
```
Full-bleed image or dark canvas
Massive headline (Instrument Serif)
One-line subtitle
Optional: single CTA
Gradient fade at bottom → next section
```
*Used when:* Opening a page. Making a first impression.

### Feature Grid Pattern
```
3-column grid of cards
Each card: icon/illustration → heading → description
Equal weight, equal size
Staggered reveal on scroll
```
*Used when:* Presenting multiple capabilities, values, or features.

### Bento Layout Pattern
```
Asymmetric grid of cards
Cards vary in size (1×1, 2×1, 1×2, 2×2)
Used for visual hierarchy — bigger cards = more important
```
*Used when:* Showcasing work with varying importance. Portfolio highlights.

### Split Layout Pattern
```
50/50 or 60/40 split
Text on one side, image on the other
Alternates on repeat rows
Collapses to stack on mobile
```
*Used when:* Alternating between narrative and visual. Case studies. About pages.

### Editorial Pattern
```
Narrow reading column (max 720px)
Serif headlines
Generous whitespace
Full-bleed image breaks between text blocks
```
*Used when:* Long-form storytelling. Manifestos. Founder letters.

### Timeline Pattern
```
Central vertical line
Alternating left/right entries
Year markers as anchors
Staggered reveal from alternate sides
```
*Used when:* Showing progression. Journey. Growth over time.

### Comparison Pattern
```
Two-column split
Left: Before / Problem. Right: After / Solution.
Visual contrast between sides
```
*Used when:* Showing transformation. Before/after. Problem/solution.

### Close Pattern
```
Centered, tight, focused
One clear call to action
Generous whitespace
Feels final — like the last page of a book
```
*Used when:* Ending a page. Inviting action. Making the final impression.

---

## Pattern Rules

### 1. Use existing patterns
Before inventing a new layout, check if an existing pattern solves the problem. Patterns are battle-tested. New layouts are experimental.

### 2. Patterns are combinable
A page can use multiple patterns: Hero → Editorial → Feature Grid → Close. Patterns snap together like LEGO. The sequence creates the narrative.

### 3. New patterns must earn their place
If a project requires a genuinely new layout, it must prove itself across at least two more projects before becoming a pattern. One use is an experiment. Three uses is a pattern.

---

*Patterns are not a lack of creativity. They're creativity, crystallized.*
