# 022 — Templates

## Philosophy

**Templates are starting points. Not shortcuts.**

Every person is unique. Every digital identity should be too. But there are patterns — narrative structures, layout rhythms, section sequences — that recur across projects. Templates capture these patterns as starting points. They are never delivered as-is. They are always adapted to the individual.

---

## Template Library

### 000 — The Creator
Single-page narrative identity. Hero → Philosophy → Work → Journey → Connect.
*For:* Individual content creators, artists, writers.

### 001 — The Founder
Multi-page identity. Home → About → Work → Story → Contact.
*For:* Entrepreneurs, indie makers, startup founders.

### 002 — The Brand
Editorial lookbook. Hero → Collection → Philosophy → Products → Newsletter.
*For:* Fashion brands, design studios, lifestyle brands.

### 003 — The Portfolio
Work-first showcase. Hero → Selected Work (grid) → Process → About → Contact.
*For:* Designers, photographers, agencies.

### 004 — The Cinematic
Dark, dramatic narrative. Loading → Hero → Journey → Craft → Numbers → Connect.
*For:* Athletes, entertainers, high-impact personal brands.

### 005 — The Startup
Product-led with personality. Hero → Problem → Solution → Features → Team → CTA.
*For:* Early-stage startups, SaaS, tech companies.

---

## Template Rules

### 1. Templates are skeletons
A template provides structure — section order, layout rhythm, narrative arc. It does not provide design. Color, type, motion, photography — these come from the client.

### 2. Templates are never delivered unchanged
Every template is adapted. Sections are added, removed, reordered. The template is the starting line, not the finish.

### 3. Templates evolve
Every project contributes back to the template library. A section that worked beautifully for one client may become a pattern for others.

---

*The template is the map. The client is the territory. The territory always wins.*
