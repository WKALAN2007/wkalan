# 06_Knowledge_Base

## WKALAN 的圖書館。

這裡不是文件。是長期累積的知識。每一次研究、每一個靈感、每一份分析——都存在這裡，讓未來的設計判斷越來越成熟，而不是每次都重新開始。

---

## 結構

```
06_Knowledge_Base/
├── Brand/           ← Apple, Muji, Nike...品牌如何建立
├── Design/          ← Dieter Rams, Editorial Design...設計原則
├── Technology/      ← Next.js, framer-motion...技術趨勢
├── Business/        ← 定價策略、客戶體驗、商業模式
├── Psychology/      ← 認知心理學、情感設計、行為經濟
├── Storytelling/    ← 敘事結構、文案技巧、品牌故事
├── Photography/     ← 攝影風格、光線、構圖
├── Typography/      ← 字體研究、排版系統
├── Research/        ← 競品分析、行業報告、使用者研究
└── Inspiration/     ← 網站、雜誌、書、電影、展覽
```

---

## 使用方式

### 研究時
遇到一個問題（例如：定價策略）→ 打開 `Business/` → 看看別人怎麼做 → 形成自己的判斷。

### 設計時
需要靈感（例如：Chinese typography）→ 打開 `Typography/` → 參考研究 → 應用在專案。

### 成長時
每完成一個專案 → 把學到的東西寫下來 → 放進對應的資料夾 → 知識累積。

## 一份好的研究筆記

```
# [主題名稱]

## 核心觀點
(1-3 句話總結最重要的洞察)

## 詳細分析
(深入說明)

## 對 WKALAN 的啟發
(我們可以怎麼應用)

## 來源
(連結、書名、作者)
```

---

## 原則

**不是收集。是提煉。**

不要存一堆連結說「以後再看」。
每一次存進 Knowledge Base 的東西，都應該經過你的思考：
「這對 WKALAN 有什麼啟發？」

---

*知識不累積，永遠在重新開始。*
